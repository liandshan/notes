查看Controller实现

在Rails项目中，Controller用来接收和响应用户请求，后面会详细讲解，这样先简单查看学习下。前面创建了Article模型，脚手架就会创建对应的Controller，默认名称为articles_controller.rb。

文章Controller

#创建一个类,继承ApplicationController
class ArticlesController < ApplicationController
  #回调,后面会讲解
  
  #回调名称,调用方法,哪些方法生效
  before_action :set_article, only: [:show, :update, :destroy]

  #主页,对应的API地址为:/articles,get请求方式
  # GET /articles

  #这里是定义了一个方法,是Ruby语法,如果不熟悉
  #请先学习Ruby
  def index
    #Article是模型名称
    #all方法会查询所有对象
    #并赋值到@articles变量上
    
    #Ruby命名规则:@开始是实例变量; @@开始是类变量,不加任何符号是局部变量
    @articles = Article.all

    #将@articles列表渲染为json格式
    render json: @articles
  end

  # GET /articles/1
  def show
    #这里的@article,其实是通过前面的回调来初始化的
    render json: @article
  end

  # POST /articles
  def create
    #article_params是定义的一个方法
    #返回一个字段,包含用户提交的数据
    
    #在创建一个Article,并用用户传递的数据初始化
    @article = Article.new(article_params)

    #调用保存方法
    #失败返回false
    if @article.save
      #保存成功,渲染@article对象为json,http状态为created,值为201
      #当然返回200也是没问题的,但返回201更符合HTTP规范
      #可以从这里看出来,Rails框架时一个严谨的框架
      render json: @article, status: :created, location: @article
    else
      #保存失败
      #将错误渲染为json返回,HTTP状态为422
      render json: @article.errors, status: :unprocessable_entity
    end
  end

  # PATCH/PUT /articles/1
  def update
    if @article.update(article_params)
      render json: @article
    else
      render json: @article.errors, status: :unprocessable_entity
    end
  end

  # DELETE /articles/1
  def destroy
    @article.destroy
  end

  #定义私有的方法
  private
    #回调调用的方法
    # Use callbacks to share common setup or constraints between actions.
    def set_article
      @article = Article.find(params[:id])
    end

    #从请求中选择需要的参数
    #好处是只从请求中选择需要的参数,不会被黑客利用
  
    #利用用户上有一个字段,表示用户的余额,如果将所有请求的数据用来创建/更新用户
    #那个这里就有一个漏洞,黑客可以传递任何值,更改该字段
    # Only allow a trusted parameter "white list" through.
    def article_params
      params.require(:article).permit(:title, :content)
    end
end

其他的Controller内容基本上和他差不多，但企业开发中肯定有很多重复的，后面我们做项目时会讲解。
